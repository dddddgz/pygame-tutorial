Sprite（1） ​

Pygame 里有一个叫做 Sprite（动画精灵）的概念（我觉得不必翻译，很多专业术语翻译成中文就很奇怪）。通常，Sprite 拥有 image 和 rect 属性，前者为一个 pygame.Surface（pygame.image.load(path) 就返回这个类型），后者为 pygame.Rect，它可以方便的控制与计算 Sprite 的位置。

我们将在接下来的程序里使用 pygame.Rect 确定球应该在哪里，设置球碰到边缘会回弹。

我们先不必使用 Sprite 的概念，先尝试用 pygame.Rect 使球回弹。

pygame.Rect 可以使用 surface.get_rect() 获得，默认左上角为 (0, 0)。

python
import pygame
pygame.init()
SIZE = WIDTH, HEIGHT = (640, 480)
screen = pygame.display.set_mode(SIZE)
image = pygame.image.load("beach_ball.png")
rect = image.get_rect()
velocity = [10, 5]
clock = pygame.time.Clock()
running = True
while running:
    clock.tick(30)
    rect.move_ip(velocity)
    if rect.right >= WIDTH or rect.left <= 0:
        velocity[0] = -velocity[0]
    if rect.bottom >= HEIGHT or rect.top <= 0:
        velocity[1] = -velocity[1]
    screen.fill((0, 0, 0))
    screen.blit(image, rect)
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False
    pygame.display.flip()
pygame.quit()

我们设置了 velocity 变量表示球的速度，velocity[0] 为 x 速度，velocity[1] 为 y 速度。

其中，rect.move_ip(step) 可以移动 rect，step 为 (dx, dy) 或它的 list 形式。

rect.move(step) 与 move_ip 类似，但 move 是返回新的 rect，而不是直接在原来的 rect 上修改。

接下来，我们将加入总共十个球，把 screen 变大，观察它们之间的碰撞。我们将使用 Group（Sprite 组）的概念，检测球之间的碰撞。

python
import pygame
import random
pygame.init()

class Ball(pygame.sprite.Sprite):
    def __init__(self, center):
        super().__init__(group)
        self.velocity = [random.randint(5, 10), random.randint(5, 10)]
        self.image = pygame.image.load("beach_ball.png")
        self.rect = self.image.get_rect(center=center)

    def bounce(self):
        self.velocity[0] = -self.velocity[0]
        self.velocity[1] = -self.velocity[1]

    def move(self):
        self.rect.move_ip(self.velocity)
        if self.rect.left <= 0:
            self.velocity[0] = abs(self.velocity[0])
        if self.rect.right > WIDTH:
            self.velocity[0] = -abs(self.velocity[0])
        if self.rect.top < 0:
            self.velocity[1] = abs(self.velocity[1])
        if self.rect.bottom > HEIGHT:
            self.velocity[1] = -abs(self.velocity[1])
        group.remove(self)
        if pygame.sprite.spritecollide(self, group):
            self.bounce()
        group.add(self)

def add_balls(*centers):
    for center in centers:
        Ball(center)

SIZE = WIDTH, HEIGHT = (1000, 750)
screen = pygame.display.set_mode(SIZE)
group = pygame.sprite.Group()
add_balls((100, 100), (100, 200), (100, 300), (100, 400), (100, 500))
add_balls((100, 600), (200, 100), (200, 200), (200, 300), (200, 400))
clock = pygame.time.Clock()
running = True
while running:
    clock.tick(30)
    screen.fill((0, 0, 0))
    group.draw(screen)
    for ball in group:
        ball.move()
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False
    pygame.display.flip()
pygame.quit()

如果你仔细观察，就会发现有时候球之间距离很近但没有碰撞也会反弹，这是因为 Pygame 利用了 Sprite 的 rect 属性检测碰撞，Pygame 并不知道这是一个球。

Pygame 内置且常用碰撞检测方法有这些：

  • collide_circle(left, right) -> bool -> 检测两个带有 radius 属性的 Sprite 的碰撞
  • collide_rect(left, right) -> bool -> 检测两个 Rect 的碰撞
  • groupcollide(groupa, groupb, dokilla, dokillb) -> dict -> 检测两个 Group 的碰撞，dokill 表示是否从 Group 中移除 Sprite，返回 {spritea1: [spriteb1, spriteb2, ...], {sprite2: [spriteb3, spriteb4, ...]}}
  • spritecollide(sprite, group, dokill, collided) -> bool -> 检测 sprite 和 group 的碰撞，dokill 表示是否从 group 中移除碰撞的 Sprite，collided 表示一个检测两个 Sprite 是否碰撞的函数（例如 lambda s1, s2: s1.image.get_size() == s2.image_get_size()），所以这个函数不一定要被用在碰撞检测上。
